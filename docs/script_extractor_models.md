# Модели Gemini для экстракции сценария (Script Extractor)

Для задачи автоматической разбивки сырого текста книги на сцены с ролями (извлечение реплик, определение говорящего и подбор эмоции) необходимо выбрать оптимальную текстовую модель.

## Требования к задаче:
1. Огромное контекстное окно (чтобы загрузить главу целиком).
2. Поддержка строгого структурированного вывода (JSON / Structured Outputs).
3. Высокая скорость и низкая стоимость.
4. Задача заключается в **маппинге** (поиск прямой речи и привязка к словарю существующих персонажей) и базовом **анализе тональности** (определение эмоции для `prompt_override`).

## Сравнение доступных моделей

| Модель | Назначение | Подходит ли нам? |
| :--- | :--- | :--- |
| **Gemini 3.1 Pro** | Сложные логические задачи, долгосрочное планирование, глубокий анализ кода. | ❌ **Избыточна.** Модель будет слишком медленной и дорогой для простого парсинга текста. Глубокое рассуждение здесь не требуется. |
| **Gemini 3.5 Flash (Thinking: High/Medium)** | Агентные воркфлоу, написание кода с самопроверкой, сложные математические задачи. | ❌ **Избыточна.** Режим "Thinking" заставляет модель строить цепочки рассуждений перед ответом, что сильно увеличивает задержку (latency) и стоимость, не давая прироста качества для парсинга диалогов. |
| **Gemini 3.5 Flash (Standard / Low Thinking)** | Быстрая обработка огромных массивов текста, извлечение данных, форматирование в JSON, мультимодальные задачи. | ✅ **Идеальный выбор.** Поддерживает контекст свыше 1 млн токенов. Не тратит время на лишние "рассуждения", мгновенно и дешево превращает сырой текст в JSON-массив нужного формата. |

## Вывод
Для фичи "ИИ-Сценарист" мы используем **`gemini-3.5-flash`** со стандартными настройками (без глубокого Thinking). Эта модель справится с задачей быстрее всех, не сожжет бюджет API, и благодаря поддержке *Structured Outputs* гарантированно вернет валидный JSON для нашего бэкенда.
